You instruct gWLM how to manage your workload by associating, or applying, a policy to a workload. A policy includes information on the minimum amount of CPU and maximum amount of CPU the policy will request for a workload. Also, it indicates when gWLM should adjust a workload's resource allocation.

In this step:

  1. For each workload:

    1. Enter or change the name of the workload to a meaningful name to simplify working with it later. Legal characters for the name are A-Z, a-z, 0-9, the dash character ( - ), the period ( . ), and the underscore character ( _ ); the first character must be a letter.

    2. You must select a policy for the workload.

      See the bottom of the gWLM window on which you associate the policies with workloads for descriptions of the various policies. The policies included with gWLM are listed first, followed by descriptions for any policies you create.

  2. Select the [Next] button.

Creating new policies

If you do not see any policies you want to use, you can create your own policies using one of two strategies:

  • Create policies before starting the Manage New Systems wizard:

    1. Back out of the wizard (by repeatedly selecting the [Prev] button) if need be

    2. Create policies using the menu item

      Optimize Global Workload Manager (gWLM) Edit Policies

    3. Start the Manage New Systems wizard again

      Optimize Global Workload Manager (gWLM) Manage New Systems

    4. Use the newly created policies when prompted to associate policies with workloads

  • Use gWLM-supplied policies to finish the Manage New Systems wizard then create policies

    1. Select a gWLM-supplied policy for each workload

    2. Complete the Manage New Systems wizard

    3. Create the new policies using the menu item

      Optimize Global Workload Manager (gWLM) Edit Policies

    4. Associate the new policies with your workloads using the menu item

      Optimize Global Workload Manager (gWLM) Edit Associations
