The Associations tab allows you to:

» Associate a policy to multiple workloads
» Change the policy for a workload
» Delete policy-workload associations

The Associations tab provides a table that includes the following columns:

ColumnDescription
Policy Name of a policy
Workload The workload associated with the policy (One policy can be associated with multiple workloads.)
Shared Resource Domain The SRD in which the workload is running
Hostname The host where the SRD is

Using the table:

  • Re-sort the rows by selecting the column heading that you would like to sort by.

  • Limit the associations displayed using the pull-down above the table labeled List associations for policy.

Associate a policy to multiple workloads

Associate, or apply, a policy to a workload to have that policy influence gWLM's resource allocation to the workload.

To associate a policy to one or more workloads:

  1. Select the following menu item if you have not already done so:

    Optimize Global Workload Manager (gWLM) Edit Associations

  2. Select the [Associate Policy to Workloads] button.

  3. Scroll down to the new section "New Association" of the window.

  4. Select the policy that you want to associate using the pull-down labeled Select policy.

    Once you select a policy, the Available workloads list displays the workloads to which you can apply the policy.

  5. Select one or more workloads from the Available workloads list to associate with the policy.

    To select a range of workloads, select a workload, hold down SHIFT, then select the workload on the other end of the range.

    To select multiple, separated workloads, hold down CTRL and select the various workloads.

    To unselect a workload, hold down CTRL and select the workload.

  6. Select the [ >> ] button to move the workloads to the list of Workloads to associate with the policy.

  7. Select the [OK] button.

Change the policy for a workload

A workload can have only one policy at a time.

To change a workload's policy:

  1. Select the following menu item if you have not already done so:

    Optimize Global Workload Manager (gWLM) Edit Associations

  2. Select a policy.

  3. Select the [Change Policy for Workload] button.

  4. Scroll down to the new "Edit Association" section.

  5. Select a new policy for the workload using the pull-down.

  6. Select the [OK] button.

Delete policy-workload associations

Deleting a policy-workload association leaves a workload with no policy to govern its resource allocation. Consequently, you can delete policy-workload associations only for unmanaged workloads.

To delete policy-workload associations:

  1. (Optional) Ensure all the workloads for which you want to delete associations are unmanaged. Complete this task using the Unmanage Workloads wizard, available through the menu item:

    Optimize Global Workload Manager (gWLM) Unmanage Workloads

  2. Select the following menu item if you have not already done so:

    Optimize Global Workload Manager (gWLM) Edit Associations

  3. Select the associations.

  4. Select the [Delete] button.

  5. Confirm the deletion.
