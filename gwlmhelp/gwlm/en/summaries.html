Optimize Global Workload Manager (gWLM) View Summaries

gWLM allows you to easily monitor SRDs and their workloads in real-time by selecting the View Summaries menu item.

This help page covers the following topics:

» Summary of SRDs table
» Individual SRD detail table
» Status information: Critical, Major, Minor, and Normal

Sort the rows in a table by selecting the column heading that you would like to sort by.

Summary of SRDs table

The main table in the View Summaries window includes the following columns:

ColumnDescription
Shared Resource Domain

Name of an SRD

Select an SRD to view data on its workloads. Scroll down to see this data.

TypeThe type of compartment managed in the SRD: npar, vpar, pset, or fss group
ModeManaged / Advisory
Size

The amount of a resource a compartment actually has

For CPU, size may differ from the allocation when gWLM is deployed in advisory mode.

Utilization

The percentage resulting from dividing the amount of a resource consumed in the SRD by the amount of the resource available in the SRD

Last UpdateThe time at which the data being presented was gathered
HostnamesHosts with workloads in the SRD

Individual SRD detail table

When you select an SRD in the first table on the View Summaries window, a second table appears. This table shows the following data for the workloads in the SRD:

ColumnDescription
Workload

Name of a workload

Select a workload to go to the View Real-time Reports window, where you can graph workload data.

Policy

Name of the policy associated with the workload

Select a policy to go to the View Real-time Reports window, where you can graph policy data.

Policy Status

Status showing how well the policy is being met

For information on the various status icons, see the table below.

RequestThe amount of a resource the policy is requesting for the workload
Allocation

The amount of a resource, such as CPU, that gWLM sets aside for a compartment after arbitrating resource requests from the policies for all the compartments

In managed mode, gWLM makes an allocation available to a compartment. In advisory mode, however, gWLM simply reports what the allocation would be--without actually affecting resource allocations on a system.

The amount allocated may be restricted by the minimum and maximum number of CPUs you specify for compartments when creating them using npar, vpar, or pset commands.

Size

The amount of a resource a compartment actually has

For CPU, size may differ from the allocation when gWLM is deployed in advisory mode.

Utilization

The percentage resulting from dividing a workload's consumption of a resource by its allocation of that resource.

For example, assume a workload has an allocation of 5 CPUs. If the workload is consuming 4.5 CPUs, its utilization percentage is 4.5/5, or 90%.

HostnameHost where the workload is running

Status information: Critical, Major, Minor, and Normal

This section explains what to do for a given status.

StatusPossible causeAction
Critical

  • A daemon is not running

  • Network issues

  • A managed node is down

  • Ensure gwlmagent is running on each managed node

  • Check the network

  • Ensure each managed node is up

Major

An SRD is in managed mode and has one or more of the following conditions:

  • Request > Size

  • Allocation > Size

  • Request > Allocation

  • Request > Size or Request > Allocation:

    There are not enough resources to meet the request. For a utilization policy, this means the policy is not being satisfied. For an OwnBorrow policy, the policy is most likely being satisfied because the workload is getting what it owns (the allocation); however, the request for additional resources (request for borrowing) is not being met.

  • Allocation > Size:

    The CPU count has been decreased manually, outside gWLM. If this status persists beyond one interval, contact HP support.

Minor

An SRD is in advisory mode and has one or more of the following conditions:

  • Request > Size

  • Allocation > Size

In both cases: If there are excess resources that can be allocated to the workload, using gWLM in managed mode would be beneficial.

NormalData is current and policies are being satisfiedNo action needed
